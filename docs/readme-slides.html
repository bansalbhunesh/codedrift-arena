What’s wrong?

The codebase already changed (rename, delete, new contract)…

…but the PR still calls the old API. Looks fine in isolation.

Tests fail for real. That’s the ground truth.

One loop

  1. Generator plants a change.
  2. A PR still references the stale world.
  3. Pytest runs → failure output is shown to the model.
  4. Reviewer must name the bug, trace the path, block the merge.
  5. Scorer returns reward → GRPO updates the policy.

Four pieces

Generator — mutates a mini-repo (AST-level patterns).
Execution — real test run; failing output = oracle.
Reviewer — LLM; structured report (verdict, root cause, path…).
Scorer — causal, anti-hallucination, calibration.

Judges: one theme

Theme #3.1 — World modeling (professional tasks):
the agent must hold a consistent world model of a changing codebase, use tool-grounded evidence (tests), and get rewarded for correct causal beliefs — not for fluent approval.
